🔐 🔐 Integridad de Código (4/4 validados)

Verifica que el código no ha sido modificado maliciosamente. Cada versión del código tiene un hash único SHA-256 que se registra en Fabric. Si el código cambia, el hash cambia y la verificación falla.

✅ Registro de código con hash SHA-256
El código se registra con su hash para futuras verificaciones
Ejemplo: register_code(['master.py'], '1.0.0')
✅ Verificación pasa para código sin modificar
Si el código no ha cambiado, la verificación es exitosa
Ejemplo: verify_code(['master.py']) → True
✅ Verificación falla para código modificado
Si el código fue alterado, se lanza CodeIntegrityError
Ejemplo: verify_code() → CodeIntegrityError
✅ Auto-verificación de código propio
Cada componente verifica su propio código antes de procesar
Ejemplo: verify_own_code() → True
💡 Uso: security.register_code(['mi_script.py'], '1.0.0') # Registrar
security.verify_code(['mi_script.py']) # Verificar

🔑 🔑 Firmas ECDSA (3/3 validados)

Criptografía de curva elíptica para firmar mensajes. Cada identidad tiene una clave privada (en el MSP) que usa para firmar, y una clave pública (en el certificado) que otros usan para verificar.

✅ Firma con clave privada ECDSA
La firma se genera usando la clave privada del MSP
Ejemplo: sign('datos', 'CN=Master') → 'base64:signature'
✅ Verificación con certificado público
Se verifica usando la clave pública del certificado
Ejemplo: verify_signature(data, sig, 'CN=Master') → True
✅ Firma inválida rechazada
Si la firma no corresponde al datos, la verificación falla
Ejemplo: verify_signature(...) → False
💡 Uso: signature = gateway.sign(hash, signer_id) # Firmar
gateway.verify_signature(hash, sig, signer_id) # Verificar

🛡️ 🛡️ Permisos de Comunicación (4/4 validados)

Control de acceso que define quién puede comunicarse con quién. Zero Trust significa que nunca se confía automáticamente en una comunicación, siempre se verifica el permiso.

✅ Registro de permiso de comunicación
Se registra que A puede enviar a B
Ejemplo: register_communication('CN=Master', 'CN=Slave')
✅ Permiso concedido retorna True
Si existe el permiso, la verificación es exitosa
Ejemplo: can_communicate_with('CN=Master', 'CN=Slave') → True
✅ Permiso denegado retorna False
Si no existe el permiso, se deniega la comunicación
Ejemplo: can_communicate_with('CN=Unknown', 'CN=Slave') → False
✅ Agregar participante confiable
Se registra un participante con sus permisos asociados
Ejemplo: add_trusted_participant('CN=Master', ['CN=Slave'])
💡 Uso: # Registrar que Master puede hablar con Slave
security.register_communication('CN=Master', 'CN=Slave')
# Verificar antes de procesar
security.can_communicate_with('CN=Master', 'CN=Slave')
